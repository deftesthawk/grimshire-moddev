Adding references

In this step, you will add the required game, Unity, and framework DLLs to your project as references.

Why references matter

References are links to external assemblies that make their classes and methods available in your code. A useful way to think about it: your project is a workshop, and references are the tools you bring into it, you can only use what is actually on the bench. If a required reference is missing, Visual Studio will report unresolved types and the project will fail to build.

Add references in Visual Studio

In Solution Explorer, under your project, right click References, then select Add Reference.

The Reference Manager will open. Select Browse.

Then add the files below, located in your Grimshire install path.

Core references

Filename Location
Assembly-CSharp.dll Grimshire\Grimshire_Data\Managed
UnityEngine.dll Grimshire\Grimshire_Data\Managed
UnityEngine.CoreModule.dll Grimshire\Grimshire_Data\Managed
UnityEngine.UI.dll Grimshire\Grimshire_Data\Managed
BepInEx.dll Grimshire\BepInEx\core
0Harmony.dll Grimshire\BepInEx\core

After adding the core references, your References list should look similar to the example below. Only the entries with a checkmark are actively referenced by this project. This view may also show additional assemblies that have been used recently, making them easier to add when needed.

Optional framework or library references

Add references for any extra frameworks you actually use. For example, if your mod depends on another plugin or shared library, add that DLL as a reference too.

Dependency attribute example

If your mod depends on another plugin at runtime, declare it in your plugin class: 

[BepInPlugin(GUID, MODNAME, VERSION)]
[BepInDependency("com.example.dependency")]
public class MyPlugin : BaseUnityPlugin
{
    // ...
}

Practical note

You will not need every reference for every mod. Start with the core set, then add any additional references your project requires. As your mod evolves, your reference list will likely change.
