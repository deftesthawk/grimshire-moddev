Configuring the assembly

This section covers configuring your project settings and build options for reliable DLL output and deployment.

Project properties

In Solution Explorer, under your project double click Properties.
Alternatively you can right click your project and select Properties.

If you need a quick refresher: a solution organizes your projects, and the project is what compiles into your mod DLL.

Assembly name and namespace

In Properties, on the Application tab, review Assembly name and Default namespace. The namespace helps keep your mod code isolated and reduces naming conflicts.

To keep it unique, include your online name or identity before the mod name. For example, use a pattern like YourName.Grimshire.ModName instead of only ModName. Keep this naming pattern consistent across your project and future mods.

Build Events

You can automatically copy your compiled DLL into the Grimshire plugins folder after each successful build.
In Build Events, add a post-build command similar to this: 

mkdir "<install-path>\Grimshire\BepInEx\plugins\$(TargetName)"
copy /Y "$(TargetPath)" "<install-path>\Grimshire\BepInEx\plugins\$(TargetName)"
Replace <install-path> with the path to where Grimshire is installed on your machine

This does two things:

  • Creates a plugin folder for your mod if it does not already exist
  • Copies your built DLL into that folder

Build and verify

  1. Right click your project and select Build
  2. Check the Output window for errors
  3. Confirm your DLL appears in Grimshire/BepInEx/plugins/<YourModName>
