Creating a project

This section shows how to create a new project and keep your solution structure tidy.

Creating a Visual Studio C# project

Open Visual Studio, create a new project and select the Class Library (.NET Framework) template for C#.

Choose the correct framework

Give the project a name (for example, MyFirstMod). This name will also be used as the folder that contains all files for that mod. Set the solution name to something like Mods; this will be the folder that contains all of your Grimshire mod projects. Make sure you untick the option to place the solution and project in the same directory. Finally, set the framework to .NET Framework 4.7.2.

Solutions and projects

Visual Studio uses two concepts that matter here: solutions and projects. A solution is a container that can hold many projects. A project contains the code and configuration for a single build output, in our case, a mod DLL.

Following the steps above will create a tidy layout where each mod has its own folder, all managed under a single solution: 

Mods/
  MyFirstMod/
  MySecondMod/
  Mods.sln

You should now have

  • A solution file (.sln) that can contain multiple mod projects
  • A project that you will later build into a DLL (your mod assembly)
